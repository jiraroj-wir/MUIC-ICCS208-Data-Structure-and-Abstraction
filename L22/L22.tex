\section*{Problem 1}

1.You have a coin that turns up heads with probability p. How many tosses do you need in expectation before you encounter the pattern HT consecutively (including the tosses HT themselves)?

\medskip
\solution

let

\[E \equiv \text{number of tosses from the start before you encounter the pattern HT consecutively}\]

as $E$ is starting pattern (empty) or when the previous toss is a T --- so no partial match yet, we defined

\[{E^'} \equiv \text{expected T, comes after that partial match (H)}\]

so, initially, after the first toss (you tosses once), we got probability $p$ of landing a H, and $(1-p)$ of landing a T. we'll got a relation of

\[E = 1 + p\cdot{E^'} + (1-p)E\]

format it as

\[(1-(1-p))E = 1 + p\cdot E^'\]

\[pE = 1 + p\cdot {E^'}\]

\[E = \frac{1+p \cdot E^'}{p} && \tag{1}\]

in case we got $H$ for our first toss, the two possibility is the same: probability p for H --- the state remain as partial match, probability (1-p) for T --- which gives the $HT$ pattern, so 

\[{E^'} = 1 + p \cdot {E^'} + (1-p)\cdot 0\]

we can now solve for $E^'$ as

\[(1-p){E^'} = 1 + 0\]

hence

\[{E^'} = \frac{1}{1-p}\]

and, we can substitude $E^'$ in (1) 

\[E = \frac{1 + p\cdot \left(\frac{1}{1-p}\right)}{p}\]

so, you need $\boxed{E = \frac{1}{p} + \frac{1}{1-p}}$ tosses in expectation before you encounter the pattern HT consecutively.

\section*{Problem 2}

2.A fancy search algorithm is shown below. Suppose a key k is present in the array a, which contains unique numbers. What is the running time of this algorithm in expectation?

\begin{verbatim}
def fancy_search(a: List[int], k: int) -> int:
    random_loc = pick a number between 0 and len(a) - 1 uniformly at random
    while a[random_loc] != k:
        random_loc = pick a number between 0 and len(a) - 1 uniformly at random
    return random_loc
\end{verbatim}

\medskip
\solution

the probability of picking the right number is $\frac{1}{n}$, on each trial, there's probability $p$ that `random\_loc == k`, and $(p-1)$ that it's not (use 1 trial, and the probability remain the same); we can write that out as

\[E = p\cdot1 + (1-p)(1+E)\]

where,

\[E \equiv \text{the expected number of loop's iteration/ the algorithm returns k}\]

simplify this equation

\[E = p + 1 + E - p -pE\]

hence, 

\[E = \frac{1}{p}\]

where p in this case equalse to $\frac{1}{n}$, so $E = n \in \boxed{\Theta(n)}$
