# MUIC-ICCS208-Data-Structure-and-Abstraction

## Course Overview

This course is essentially the data structure part of the well-known data structures and algorithms sequence. It focuses on data structures, abstraction, implementation details, basic asymptotic analysis, and the programming techniques needed to work with them in practice.

The course was taught by **Prof. Kanat**. From what I have heard, many seniors prefer Prof. Kanat for courses like this, and I can understand why. I enjoyed his lectures. The scope and objective of each lecture were clear, and his explanations were good. He did not just write out code line by line. He focused more on the technical side of things, while full implementation for later topics was often left to the students.

Main topics covered in the course include:
- Java basics and class mechanics
- linked structures, queues, stacks, and array lists
- loop invariants
- interfaces, inheritance, generics, and subtyping
- higher-order functions
- unit testing and exceptions
- inductive thinking
- iterators and iterables
- asymptotic performance analysis
- sorting
- disjoint sets
- priority queues
- trees and binary search trees
- graph representation and traversal
- minimum spanning tree
- randomized algorithms
- hashing and hash tables

There were 2 classes per week over 12 weeks.

## My Experience

There were both lecture exercises and assignments.

The lecture exercises usually took around an hour, sometimes longer. The assignments changed character as the term went on. Earlier in the term, they were shorter project-based assignments and also carried more points, so it was worth paying close attention to them. Around the middle of the term, the assignments shifted more toward theoretical work, invariants, code analysis, asymptotics, and time complexity, so LaTeX typesetting became more relevant. Later assignments focused more on data structure implementation and competitive-programming-style problems.

There were also **4 quizzes** and **2 mastery exams**. The mastery exams were basically the course’s midterm and final coding assessments, held around the midterm and final periods. The quizzes were more technical, paper-based exams.

For the quizzes, you got two mock quizzes: one at the start of the term, and another a few days before the actual quiz. Each quiz covered all lectures up to that point.

There was also an internal course textbook provided on the course website. Prof. Kanat usually followed and referenced that text closely. In principle, I think it would be possible to get through the course mostly by studying the textbook alone, since the course schedule and test dates were announced in advance, but it is not a short text. For me, I used a mix of attending lectures and reading the text. That worked better, because the professor usually announced the exam scope, helped with assignments, and clarified which parts or sub-parts of the book were not actually needed for the exams.

## Mastery Exams

Mastery 1 was mostly about basic Java language features and introductory implementation work.

Mastery 2, at least in my experience, felt broader than I expected. It was announced as being more about data structure implementation, but I do not think it did the best job of evaluating data structure understanding in the way I expected. Some parts felt more centered on careful Java details and implementation traps than on the broader data structures I had prepared for.

I did not do well on Mastery 2. Part of it was my own mistake. I panicked, lost time, and once one part went wrong, it hurt both my morale and time management for the rest of the exam. There was also a kind of bad flashback effect for me from an older exam experience, and that definitely did not help.

So the main warning I would give is this: **study more broadly than you expect for Mastery 2**. From my experience, the exam can be broader and less predictable than you might think.

That said, I do not think the right conclusion is that the course is unfair. More realistically, it means you should not prepare too narrowly.

## Workload and Study Advice

Even though I had already learned more than 80% of the course topics before, through earlier programming experience and previous training, I was still disappointed in my own performance in the second mastery exam. I had also been away from competitive programming for around a year and a half, and I was less experienced in Java than in some other languages. So I expected myself to do better.

Looking back, I do not think the right framing is that I “got exposed” by the course or something like that. It is more that implementation exams can go badly even when you know the material, especially if you panic or prepare too narrowly. So if you have a bad day in this course, do not over-interpret it.

In general, you do not really need anything too special beyond:
- reading the lecture notes
- reading the given textbook
- doing the lecture exercises
- completing all assignments seriously

Cheat sheets were allowed for both mastery exams, but **not** for quizzes. Since the quizzes were paper-based, that distinction matters.
